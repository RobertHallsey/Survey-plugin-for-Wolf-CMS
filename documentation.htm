Survey Plugin for Wolf CMS

Introduction

This plugin for Wolf CMS lets you create different kinds of online surveys, present them to users, and collect their responses in coma-delimited files. You can add a survey to any page, with or without other content. To create a survey, you first describe the survey in a text file, as explained later in this document.

System Requirements

To use this plugin, you need Wolf CMS 0.7.5 or later. The plugin doesn't require a database of any kind.

Files in the Package

You should have received the system in a zip file containing the following files:

  \readme.txt
  \documentation.htm
  \index.php
  \sample-survey
  \Survey.php
  \SurveyController.php
  \views\index.php
  \views\sidebar.php
  \views\documentation.php
  \i18n\en-message.php
  \i18n\sp-message.php

There may be other language files in the i18n directory. You only need the language files for the languages you intend to support. Deleting the ones you don't need will not affect the plugin's operation.

How to Install the Plugin

Unzip the zip file to the Wolf plugins directory, keeping the zip's directory structure. Then, log into Wolf CMS as an administrator. Select the Administration tab in the back end, and check the Enabled box for the Survey plugin. That's all.

How to Use the Plugin

After you create one or more survey definition files and place them in Wolf's public directory, use this code in any Wolf page,

<?php if (Plugin::isEnabled('survey')) survey('my_survey'); ?>

where 'my_survey' is the survey definition file you wish to use. The if statement is really optional but prevents errors if you disable the plugin while there are still pages that call it.

To learn about survey definition files and more, please see the online documention in the plugin tab on the back end.
